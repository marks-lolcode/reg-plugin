Management & Help Desk Manual

For Registration management and Help Desk leads. Volunteer quick cards: reg-checkin.html · merch-checkin.html

📋 Overview

The extension is a Chrome extension that runs inside Neon CRM while staff are logged in. It reads attendee and registration pages, validates eligibility, and writes check-in data back to Neon so volunteers don't type it by hand. There is no server — all settings live in Chrome's local storage on each workstation independently.

Per-workstation settings
Turning on Management Override on one machine does not affect any other machine. Configure each station before the shift starts.
Never complete a real check-in during testing
Use the Manager Debug Walk (Options → Debugging mode) against a real Neon attendee page to verify field detection. Do not run a check-in to completion on a live attendee during setup or training.

🪟 Pop-up Behavior: Automated vs Manual

Manager-only setting (visible when override is enabled).

ModeBehaviorWhen to use
Automated (default) Check-in window opens in the page as a modal, automatically on page load. Toolbar click re-opens it. Modal is draggable by its title bar and remembers its position. Recommended for all check-in stations during con — smoothest for a busy line.
Manual Nothing opens automatically. Volunteer clicks the toolbar icon to open the classic small pop-up window. Fallback if the in-page modal ever conflicts with a Neon page layout.
Account pages always use the classic pop-up regardless of this setting. Only the registration (eventRegDetails) and attendee (attendeeEdit) pages switch to modal.

🔑 Management Override

Override lets a lead push a check-in through a red (blocked) status. Use only after personally resolving the underlying issue.

Enabling it

  1. Options page → check Enable Management Override.
  2. Enter the management password. Save Options.
  3. If the password is wrong, the options don't save and an error shows.

What changes while override is active

MANAGER OVERRIDE ACTIVE
  • Every screen shows the red MANAGER OVERRIDE ACTIVE bar above.
  • Toolbar icon carries a purple M badge.
  • Red attendee rows gain an ⚠ Override — Check In → button.
  • Full red ⛔ DO NOT ISSUE BADGE detail still shows — override doesn't hide the reason, it lets you proceed past it.
[SCREENSHOT: attendee screen with override — red banner + DO NOT ISSUE BADGE + override button]

Password handling

Share the password verbally or via a password manager only
Never in email, Slack, GitHub, or any written note. The code stores only a cryptographic hash — the real password is never in the repo.
  • Rotate annually. Regenerate the hash with tools/generate-password-hash.html (run locally in Chrome only — don't upload to GitHub).
  • Paste the new hash into config.js. See ANNUAL_UPDATE_GUIDE.md.
  • Turn override off (uncheck + Save) when stepping away from a Help Desk workstation so volunteers can't issue blocked badges.

🚦 Blocking Conditions

The badge number is deliberately hidden until every red condition is cleared and every required field is filled — so volunteers can't read out a number while a problem remains. As a manager with override, you see the full detail.

Yellow conditions with a Re-check button
Only conditions with fixableOnAttendeePage: true in the config show a Re-check button. Everything else (holds, payment, wrong year) requires the attendee to go elsewhere.

📋 Notes Gates

Notes on an account or registration must be acknowledged before a volunteer can proceed — intentional friction so important notes aren't skipped.

Account notes (yellow) — read, tick "I have read and understood the note(s) above", click Proceed to Check-In →.

Registration notes (yellow) — same pattern, then Show Attendee List →.

Single registration note (📋 screen) — read, click Note Read ✓.

If a volunteer is stuck on a note screen, the checkbox isn't ticked — the Proceed button stays disabled until it is.

✅ What Happens at Check-In

When Badge Issued is clicked, the extension automatically:

  1. Downloads a badge-printer CSV to the Downloads folder (the printer queue watches that folder).
  2. Increments Number of Active Badges in Neon.
  3. Writes the attendee's name to the Non-Transferable Name field.
  4. Records pickup date & time in the next pickup slot.
  5. Submits the Neon form and returns the volunteer to the account search.
CSV download failure
If the download fails, the volunteer is told not to issue the badge and to contact the Registration Head. That's your cue to check the workstation's Downloads folder setup and Chrome download permissions.

Merch check-in

Confirm Pickup writes the current date and time into each checked item's pickup field (format MM/DD/YYYY HH:MM, 24-hour) and submits the form. The redirect returns the volunteer to account search.

🔬 Pre-Con Field Check (Debug Walk)

The extension finds Neon's custom fields by matching label text. If Neon admins rename or reorder fields, matching can silently break. Run this check before con and after any Neon form change.

When to run this
  • After Neon admins change any attendee-form field label or order.
  • Before the first shift of each con (annual sanity check).
  • After any annual update to config.js fieldLabels.

How to run the Debug Walk

  1. Options page → enable Management Override (enter password).
  2. Set Behavior to "Debugging (pre-con field check walk)". Click Save Options.
  3. Open any real attendee's account page in Neon and click the toolbar icon.
  4. The extension walks account → registration → attendee automatically, auditing each step.
  5. A report tab opens listing every step and any field-resolution problems.
⛔ Turn Behavior back to Regular when done
Leave it on Debugging and every toolbar click triggers the walk instead of a check-in. This will confuse volunteers if left on during con.

If the report flags missing or ambiguous fields, that is an IT / annual-update fix. See ANNUAL_UPDATE_GUIDE.md and DEVELOPER.MD.

🔧 Maintenance Tools

Manager-only Maintenance panel at the bottom of the Options page.

Check Neon tab(s)
Confirms the extension's scripts loaded correctly on your open Neon tabs. Use this first when "nothing happens" on a page. If no script responds, reload the Neon tab.
Clear cached scrape data
Wipes cached page reads (attendee, registration, account) without touching settings. Use when a page shows stale data; reload the Neon tab afterward to re-scrape.
Storage contents
Expandable raw dump of everything in local storage. For diagnosing issues with IT.
Config check panel (always visible)
Validates config.js. Red errors must be fixed before con. Yellow warnings should be reviewed.

For broader problems see TROUBLESHOOTING.md.

📞 Escalation & Annual Update

Call IT when:

  • Config check or Debug Walk reports errors you can't resolve by re-running.
  • A condition fires that doesn't match reality (e.g. everyone shows "Unknown ticket") — usually means Neon field/ticket names changed.
  • Badge CSV won't download on a correctly-set-up station.
  • Extension doesn't load or scripts don't respond after reloads.

Point IT to DEVELOPER.MD and TROUBLESHOOTING.md.

The yearly update

Only edit config.js for the annual refresh
